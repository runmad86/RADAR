# RADAR

RADAR er en kamp-laeser og signalapp til fodbold.

## Kernen
- 1 kamp -> 1 tendenslaesning -> flere mulige markeder
- kamp-profil foerst, marked bagefter
- symbolhierarki foerst, tekst bagefter
- brutal aeerlighed om usikkerhed og `ingen klar case`

Foerste profiler i kernen:
- open
- hot
- locked
- none

Naeste profil under opbygning:
- pressure

## Samarbejds-DNA
- ChatGPT bygger, strukturerer og driver den konkrete udformning frem
- brugeren baerer mavefornemmelse, casesans og menneskelig retningsfornemmelse
- begge styrer retning og ideer

## Byggeprincip
RADAR skal bygges rigtigt foerst, ikke bredt foerst.

Den rigtige byggeorden er:
1. fakta-base
2. sync-base
3. summaries
4. motor
5. read-lag
6. app-visning
7. kalibrering
8. udvidelser
